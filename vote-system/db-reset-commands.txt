RESET DATABASE - Commands & Instructions

1) RECOMMENDED: Run inside Render (SQL Editor)
- Open Render dashboard → Databases → your database → "Open SQL" or "SQL Editor".
- Paste the full SQL from `db/postgresql/reset_database.sql` and execute.
- If you want to reseed, paste and execute the contents of these files in order:
  - db/postgresql/seed_candidates.sql
  - db/postgresql/seed_electors.sql
  - db/postgresql/seed_elector_passwords.sql

(Using the Render SQL Editor avoids network/port issues.)

4) If network times out (Connection timed out)
- Use Render Dashboard → Database → Open SQL Editor and paste the SQL manually (step 1).
- Or run the commands from a machine that can reach the Render DB (cloud VM, GitHub Actions, etc.).

---
5) Notes
- Order matters: reset_database.sql then seeds.
- Reset script clears votes and sets election to CLOSED (does not delete candidates/electors).
- If you want a full fresh import, run reset then the three seed files.
